Game Architecture

How Math or Die is structured — Section 1 of 3

Game State Machine

The game moves through these phases in sequence each round

FRONT PAGE
Grade select + Go
→
SETUP PHASE
60s · Stack stats
→
COMBAT PHASE
Stats compared · HP adjusted
→
ROUND END
Stats reset · Next round
→
GAME OVER
Win or lose

File Structure

src/
  components/
    GamePage.jsx ← root game screen
    FrontPage.jsx ← grade select + go button
    StatBar.jsx ← reusable for all 8 stat bars
    HealthBar.jsx ← player + enemy HP
    NumPad.jsx ← number input pad
    ActionButton.jsx ← Attack / Shield / Magic / Aura
    MathProblem.jsx ← displays current question + input
    PotionPanel.jsx ← 6 potions with counts
    Timer.jsx ← 60s countdown
  hooks/
    useGameState.js ← all game state + phase logic
    useEnemyAI.js ← enemy stat-filling behaviour
    useMathEngine.js ← generates problems by grade
  assets/ ← SVG art from MATH_OR_DIE.ai
  App.jsx ← routes between FrontPage / GamePage

Core Game State (useGameState)

Player
  • health (0–20)
  • attack / shield / magic / aura (0–15)
  • potions: { attack, shield, magic, aura, slow, heal } = 1 each
  • activeQuestion + userInput
Enemy
  • health (0–20)
  • attack / shield / magic / aura (0–15)
  • fillRate (ticks per second, scales with round)
  • slowedUntil (timestamp when Slow potion active)
Shared
  • phase: 'setup' | 'combat' | 'roundEnd' | 'gameOver'
  • timeRemaining (60 → 0)
  • grade (1–4)
  • round (1, 2, 3…)
