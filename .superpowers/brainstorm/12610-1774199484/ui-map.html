Game Page Layout

PLAYER_CHAR + PLAYER_HEALTH

Player character sprite + HP bar (20 segments). HP reduces permanently across rounds.

ENEMY_CHAR + ENEMY_HEALTH

Enemy character sprite + HP bar (20 segments). Shows "slowed" indicator when Slow potion active.

PLAYER_STATS

4× StatBar rows:
⚔ PLAYER_STAT_ATTACK (red)
🛡 PLAYER_STAT_SHIELD (blue)
✨ PLAYER_STAT_MAGIC (purple)
🌟 PLAYER_STAT_AURA (gold)
Each bar: 15 slots, fills left→right

TIMER

60s countdown.
Pulses red
under 10s.

ENEMY_STATS

4× StatBar rows (mirrored):
⚔ ENEMY_STAT_ATTACK (red)
🛡 ENEMY_STAT_SHIELD (blue)
✨ ENEMY_STAT_MAGIC (purple)
🌟 ENEMY_STAT_AURA (gold)
Fills autonomously via AI tick

BUTTON_ATTACK / SHIELD / MAGIC / AURA + MathProblem panel

4 action buttons stacked vertically. Tapping activates the button and shows the math question in the panel to its right. Active button highlights. Only one active at a time.

POTIONS (POT_ATTACK/SHIELD/MAGIC/AURA/SLOW/HEAL)

6 potion slots. Each shows count (1). Tapping triggers math question (same as buttons but stat +3). Greyed out when count = 0. Consumed on correct answer.

NUM_PAD

0–9 + backspace + confirm (✓). Large touch targets for 7-year-old fingers. Input displayed in active MathProblem panel.

iPad 10 Fit

iPad 10 is 820×1180pt (portrait). The Paper design is 1640×2359px at 2x — this is a perfect 1:1 match. The app will render at full resolution with no scaling needed.

CSS viewport: width=device-width, initial-scale=1. The game page fills the full screen, scrolling disabled.
